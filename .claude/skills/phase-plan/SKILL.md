# Skill: phase-plan

新しい Phase に着手する前に、以下の順で作業する。

## 1. 現状確認

- `STATUS.md` の「次にやること」を読む
- `TASKS.md` の該当 Phase の「完了条件」と未チェック項目を確認
- `docs/open-questions.md` の該当 Phase に関わる未確定事項を確認

## 2. 完了条件の具体化

対象 Phase について以下を明確にする。

- **成果物**: どんなファイル / 画面 / API / マイグレーションが増えるか
- **検証可能な完了条件**: どうなったら「終わり」と言えるか（例: `pnpm build` が通る、E2E が通る）
- **スコープ外**: この Phase ではやらないこと

## 3. ブレークダウン

- 1 コミット相当の粒度でタスクを分解
- 依存関係を明示
- `TASKS.md` の該当 Phase を更新（チェック項目を増減する）

## 4. リスクと open-question 整理

- この Phase で新しく浮かぶ可能性のある不確実性を予測
- 必要なら `docs/open-questions.md` に追記
- 破壊的判断が必要なら、ユーザーに確認する候補としてメモ

## 5. 実装着手

上記が整ったら、1 タスクずつ実装する。
コミットは **意味ある粒度ごと** に行い、`TASKS.md` / `STATUS.md` を更新する。

## 出力フォーマット

この skill を実行する際は、最終的に以下を提示する。

```
## Phase N 計画

**完了条件**
- ...

**スコープ外**
- ...

**タスク分解**
1. ...
2. ...

**リスク / 未確定**
- ...

**最初の 1 コミットで扱う範囲**
- ...
```
